## 寄存器

* [用户自定义寄存器](#user-defined-registers)
* [默认寄存器](#default-registers)
* [特殊寄存器](#special-registers)

在 Helix 中，寄存器是存储文本和其他数据的位置，例如搜索结果。寄存器可以用于剪切、复制和粘贴文本，类似于其他文本编辑器中的剪贴板。使用方式与 Vim 类似，用 `"` 来选择寄存器。

### 用户自定义寄存器

Helix 允许你创建自己的命名寄存器来存储文本，例如：

* `"ay` - 将当前选区复制到寄存器 `a`。
* `"op` - 将寄存器 `o` 中的文本粘贴到选区之后。

如果在执行更改或删除命令之前选择了一个寄存器，选区将被存储到该寄存器中，然后执行该操作：

* `"hc` - 将选区存入寄存器 `h`，然后进行更改（删除并进入插入模式）。
* `"md` - 将选区存入寄存器 `m` 并删除。

### 默认寄存器

使用寄存器的命令（如 yank `y`）如果未指定寄存器，则使用默认寄存器。默认寄存器如下：

| 寄存器字符 | 内容             |
| ----- | -------------- |
| `/`   | 上一次搜索          |
| `:`   | 上一次执行的命令       |
| `"`   | 上一次 yanked 的文本 |
| `@`   | 上一次录制的宏        |

### 特殊寄存器

某些寄存器在读取和写入时具有特殊行为。

| 寄存器字符 | 读取时                            | 写入时         |
| ----- | ------------------------------ | ----------- |
| `_`   | 不返回任何值                         | 所有值被丢弃      |
| `#`   | 选区索引（第一个选区为 `1`，第二个为 `2`，依此类推） | 该寄存器不可写     |
| `.`   | 当前选区的内容                        | 该寄存器不可写     |
| `%`   | 当前文件名                          | 该寄存器不可写     |
| `+`   | 从系统剪贴板读取                       | 合并并复制到系统剪贴板 |
| `*`   | 从主剪贴板读取                        | 合并并复制到主剪贴板  |

当将多个选区复制到剪贴板寄存器时，选区之间用换行符连接。如果从这些寄存器粘贴，Helix 会根据最后一次复制的来源决定：

* 如果最后一次 yanked 是 Helix 会话，则粘贴为多个选区。
* 否则，剪贴板内容会作为一个选区粘贴。
